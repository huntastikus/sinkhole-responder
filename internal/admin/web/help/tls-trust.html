Sinkholed HTTPS

TLS and trusting the CA

DNS redirection does not remove HTTPS certificate checks. A browser still asks for a certificate valid for the original hostname.

The three TLS modes

  • Disabled is the safe default. The responder does not listen for HTTPS traffic.
  • Static certificates use certificate and key pairs you provide for hostnames you legitimately control.
  • Local CA creates a short-lived certificate for each requested hostname. Every client must deliberately trust the CA first.
Lab/home only

A trusted CA can impersonate any HTTPS site to that client. Use local-CA mode only on devices you control in a lab/home network. Never distribute the CA or install it broadly, and protect its private key.

Use the TLS certificate manager to inspect the current mode, generate and download a local CA, upload static certificates, or switch modes. The responder never installs trust for you.
