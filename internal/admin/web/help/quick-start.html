From zero to blocking

Quick start

The guided setup is the fastest route. You will choose a responder address, turn on the safe default protection, and get the exact DNS settings to copy.

Set it up

  1. Enable the admin GUI. Set admin.enabled: true. The supplied container configuration already enables it on first start.
  2. Create the admin password. Open http://RESPONDER_IP:8080. A new installation sends you to the password setup page.
  3. Run the setup wizard. Choose a stable LAN address that clients using your DNS sinkhole can reach. Start with HTTP unless you deliberately need sinkholed HTTPS.
  4. Enable recommended protection. The wizard can turn on the curated recommended rulepack for you.
  5. Point blocked DNS names here. Configure AdGuard Home, Pi-hole, or another compatible DNS sinkhole to return this responder's address. See the DNS sinkhole setup guide.
  6. Verify. Load a blocked resource, then check that the dashboard request counter increases. The Tools page can preview which response a host and path would receive.

HTTPS is a separate choice: sinkholed HTTPS works only with a suitable certificate. Local-CA mode requires deliberate trust on every client. Read TLS and trusting the CA before enabling it.
