DNS redirection

DNS sinkhole setup

AdGuard Home, Pi-hole, or another DNS sinkhole can work with Sinkhole Responder when blocked names resolve to the responder's stable LAN address. The responder then returns the harmless placeholder.

AdGuard Home: choose Custom IP blocking

  1. Open the AdGuard Home admin interface.
  2. Go to Settings → DNS settings.
  3. Set Blocking mode to Custom IP.
  4. Enter the responder's stable LAN IPv4 address in Blocking IPv4.
  5. Enter the responder's LAN IPv6 address in Blocking IPv6 only if clients can actually reach it over IPv6. Otherwise leave it empty when AdGuard Home permits that.
  6. Save the settings. While testing, use a low blocked-response TTL so corrections take effect quickly.

Pi-hole: choose IP blocking

Pi-hole's default NULL mode returns 0.0.0.0 or ::, so it does not send traffic here. Use IP mode and configure Pi-hole's blocking reply address as the Sinkhole Responder address. For an IPv4-only responder, use IP_NODATA_AAAA so blocked AAAA queries do not point somewhere else.

sudo pihole-FTL --config dns.blocking.mode IP sudo pihole-FTL --config dns.reply.blocking.force4 true sudo pihole-FTL --config dns.reply.blocking.IPv4 RESPONDER_IP

Other DNS sinkholes

Use the equivalent custom-address, rewrite, or blocking-reply setting. The required behavior is simple: a blocked A or AAAA lookup must return the reachable Sinkhole Responder address instead of 0.0.0.0, ::, NXDOMAIN, or NODATA.

The responder must be reachable on port 80 for HTTP and, only when TLS is deliberately enabled, port 443 for HTTPS. DNS changes the destination address; it does not change the destination port.

Generate ready-to-paste YAML

Open Tools → AdGuard Home config, enter the responder's LAN address, and generate the matching AdGuardHome.yaml fragment. The tool only prepares text; it never connects to or changes AdGuard Home.

DNS still points elsewhere? Check the sinkhole's reply mode, browser Secure DNS / DNS-over-HTTPS, the operating system, and any VPN DNS setting. Continue with Troubleshooting.
